## 🔑 Key Optimizations

### 1️⃣ Move Semantics
```cpp
Task(Task&& other) noexcept = default;
```
**Result**: No unnecessary copying of Task objects

### 2️⃣ Vector Pre-allocation
```cpp
filtered.reserve(tasks.size());
for (const auto& task : tasks) {
    filtered.push_back(task);  // No reallocation
}
```
**Result**: Single memory allocation instead of multiple

### 3️⃣ String Views (Zero-Copy)
```cpp
void getStringInput(std::string_view prompt)
```
**Result**: No string copy when passing parameters

### 4️⃣ Exception Safety
```cpp
const std::string& getId() const noexcept;
```
**Result**: Clear contracts; enables compiler optimizations

### 5️⃣ Algorithm Improvements
```cpp
std::count_if(tasks.cbegin(), tasks.cend(), 
              [](const Task& t) { return !t.isCompleted(); })
```
**Result**: No temporary vector creation

### 6️⃣ Const Correctness
```cpp
const int totalTasks = manager.getTotalTasks();
const int activeTasks = manager.getActiveTasksCount();
```
**Result**: Cached values, prevents repeated calculations

## 🎓 Learning Resources

### For Developers Adding Features

Follow these patterns:

```cpp
// ✅ DO: Use noexcept on non-throwing methods
int getCount() const noexcept;

// ✅ DO: Use string_view for string parameters
void display(std::string_view text);

// ✅ DO: Pre-allocate vectors when size is known
std::vector<Task> results;
results.reserve(approximateSize);

// ✅ DO: Mark important return values with [[nodiscard]]
[[nodiscard]] const std::string& getId() const;

// ✅ DO: Cache repeated calculations
const int total = getTotalTasks();
const int completed = getCompletedTasksCount();

// ❌ DON'T: Create temporary vectors in loops
for (int i = 0; i < counts(); ++i) {
    std::vector<Task> temp = getTasks();  // BAD!
}
```
